## 3. 核心数据结构：多模态异构属性图 (Heterogeneous Attributed Graph)
[cite_start]摒弃简单的拓扑图，必须将物理性能与环境意图降维转化为图网络可计算的张量特征 [cite: 28, 71]。

### 3.1 节点定义 (Nodes)
[cite_start]包含空间节点与全局/环境节点 [cite: 8]。
| 特征维度 | 具体参数与代理指标 | 说明 |
| :--- | :--- | :--- |
| **基础属性** | 空间类型 (教室/走廊/楼梯等) | [cite_start]节点语义标签 [cite: 19]。 |
| **几何容限** | 预期面积要求 (㎡)、长宽比容限 | [cite_start]建筑空间的三维映射约束 [cite: 8]。 |
| **采光代理** | 对外部自然光的刚性需求等级 | [cite_start]如普通教室设为最高级，强制连接外部环境节点 [cite: 19, 151]。 |
| **动线代理** | 到达主出入口的最短拓扑步数、介数中心性 | [cite_start]用于预判学校爆发性人流的流线效率与拥堵瓶颈 [cite: 18, 19]。 |
| **环境节点** | “正南向节点”、“主干道接驳点” | [cite_start]虚拟的全局节点，用于施加宏观朝向与文脉约束 [cite: 35, 100]。 |

### 3.2 边定义 (Edges)
[cite_start]描述节点间的多维关系 [cite: 101]。
* [cite_start]**物理连通边**：代表有门/通道，直接关联消防疏散路径搜索 [cite: 101]。
* [cite_start]**声学阻断边**：代表具有强隔音属性的实体墙，满足动静分区约束 [cite: 101]。
* [cite_start]**视线/采光边**：代表视线与光线可穿透但物理不可通行（如内天井玻璃窗） [cite: 101]。

## 4. 神经符号 (Neuro-Symbolic) 约束机制
[cite_start]将建筑规范显性化为代码约束，解决概念设计期的信息稀疏性 [cite: 75, 76]。

* [cite_start]**前置规则富化 (Data Enrichment)**：通过脚本自动计算衍生参数。例如，根据规范（普通教室 1.0 - 1.5 ㎡/人）由“面积”自动推断“疏散人数” [cite: 38, 150]。
* [cite_start]**硬约束 (拓扑掩码 Topological Masking)**：绝对红线。例如：人数超过阈值的房间节点，其安全出口连边数（Degree）必须 $\ge$ 2；音乐教室与普通教室之间强加声学阻断边或要求最短路径距离 $\ge$ 2 [cite: 34, 35, 150]。
* [cite_start]**软约束 (损失函数惩罚 Loss Penalty)**：性能优化。例如：教学用房未连接到采光节点，或图的交通核面积占比过高时，施加 Loss 惩罚 [cite: 34, 154]。

    * [cite_start]构建受限 GNN 评价网络，筛选高分合规图集 [cite: 3]。

## 6. 与 Claude Code 对话的核心“护栏”指令
1. **数据结构优先**：在编写 GNN 模型前，优先保证多模态异构属性图的张量维度绝对对齐。这是后续一切数学运算的基石。
2. **测试驱动开发 (TDD)**：所有复杂图操作（最短路径计算、介数中心性更新、拓扑掩码应用）必须配备轻量级的 `test.py`。
3. [cite_start]**拒接黑盒 API**：对于建筑规范的软/硬约束公式，必须写出清晰的、可逐行断点调试的显式 Python 逻辑代码，以确保论文中对应公式的严谨性 [cite: 4]。
